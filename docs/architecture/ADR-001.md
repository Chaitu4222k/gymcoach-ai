# ADR-001 - Project Architecture

## Status

Accepted

## Decision

The project will follow a monorepo architecture.

## Structure

- client → React application
- server → Node.js + Express API
- ai-service → Python FastAPI service
- shared → Shared resources between services
- docs → Project documentation

## Reason

- Easy to maintain
- Easy to deploy
- Clear separation of responsibilities
- Suitable for future AI/ML expansion

## Consequences

- Services remain independent
- AI can evolve without affecting the backend
- Easier to explain during interviews
